Home Automation Dashboard with Structured Sensor Data

Monitoring simulated environmental conditions using PostgreSQL and Dash

Overview

This project simulates a smart home automation system that tracks and stores environmental sensor data such as temperature, humidity, wind speed, and light levels. The data is structured and stored in a PostgreSQL database, representing readings from multiple rooms and timestamps. A Flask-based web server combined with Dash and a separate Grafana server is used to create a responsive dashboard for visualizing recent sensor data and observing trends across various home conditions.

Images & Screenshots

Key Features

  • Simulated IoT sensor data representing a home environment
  • PostgreSQL backend with relational schema for time-series storage
  • Flask API for data access and integration with Dash and Grafana visualizations
  • Interactive dashboard to view metrics by room, sensor type, or time window
  • Clean modular layout for future integration with live sensor input

Technologies Used

Python, Flask, Dash, Grafana, PostgreSQL, Psycopg2, SQLAlchemy, Pandas

Challenges & Lessons

Designing a flexible schema for time-series sensor data in a relational database was a key challenge. I learned to structure queries that allow dynamic filtering and aggregation by room or time, and how to use Dash effectively for creating real-time charts. The project reinforced my understanding of RESTful APIs, Flask architecture, Grafana dashboards, and working with structured databases for IoT use cases.
