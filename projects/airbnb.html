Smart Monitoring Dashboard for Airbnb Hosts

Real-time IoT data tracking for enhanced property oversight

Overview

This project simulates an IoT system designed to help Airbnb hosts monitor their properties remotely using various environmental and occupancy sensors. The system collects data from motion detectors, audio sensors, water leak detectors, door contact sensors, temperature sensors, and air quality monitors. All data is stored in a MongoDB database and visualized through a Flask dashboard. The dashboard alerts users to recent activity and potential issues, such as leaks, loud noises, or poor air quality, making it an effective tool for managing property risks while improving guest safety and experience.

Key Features

  • Simulated sensor data from multiple IoT devices relevant to short-term rentals
  • MongoDB backend to store time-series data with JSON-like structure
  • Flask dashboard displaying real-time alerts and recent activity summaries
  • Conditional logic for generating warnings (e.g., excessive noise or temperature anomalies)
  • Modular architecture for easily extending to new sensor types or properties

Technologies Used

Python, Flask, MongoDB, Pymongo, JavaScript, HTML/CSS, Requests, Simulated IoT Data Pipelines

Challenges & Lessons

The project required careful planning around data modeling for MongoDB to efficiently store and retrieve sensor event streams. I also learned how to design alert logic to flag conditions that matter to a host, such as fire, flooding, gunshot detection, and unapproved entry. This work improved my skills in backend integration, dashboard development, and real-time data monitoring.
